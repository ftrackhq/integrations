Post Slack message on publish¶

Next, we implement a custom finaliser within Maya - a small function that posts a Slack message containing the version ident, the comment and a thumbnail (can be replaced with the full Quicktime reviewable movie if desired), with each publish made.

Functions that run after load and import are called “finalizers” and they are fed all the data from the previous steps and stages.

As a preparation, we need to have the thumbnail publisher to save a copy to be used with the Slack post:

Finaliser¶

mypipeline/projects/framework-maya/resource/plugins/python/publisher/finalisers/common_slack_post_publisher_finalizer.py

 1# :coding: utf-8
 2# :copyright: Copyright (c) 2014-2022 ftrack
 3import os
 4
 5from slack import WebClient
 6
 7import ftrack_api
 8
 9from ftrack_framework_core import plugin
10
11
12class CommonSlackPublisherFinalizerPlugin(plugin.PublisherPostFinalizerPlugin):
13    plugin_name = 'common_slack_publisher_finalizer'
14
15    SLACK_CHANNEL = 'test'
16
17    def run(self, context_data=None, data=None, options=None):
18        # Harvest publish data
19        reviewable_path = asset_version_id = component_names = None
20        for component_data in data:
21            if component_data['name'] == 'thumbnail':
22                for output in component_data['result']:
23                    if output['name'] == 'exporter':
24                        reviewable_path = output['result'][0]['result'][0]
25            elif component_data['type'] == 'finalizer':
26                for step in component_data['result']:
27                    if step['name'] == 'finalizer':
28                        asset_version_id = step['result'][0]['result'][
29                            'asset_version_id'
30                        ]
31                        component_names = step['result'][0]['result'][
32                            'component_names'
33                        ]
34                        break
35
36        # Fetch version
37        version = self.session.query(
38            'AssetVersion where id={}'.format(asset_version_id)
39        ).one()
40
41        # Fetch path to thumbnail
42        if reviewable_path:
43            # Assume it is on the form /tmp/tmp7vlg8kv5.jpg.0000.jpg, locate our copy
44            reviewable_path = os.path.join(
45                os.path.dirname(reviewable_path),
46                'slack-{}'.format(os.path.basename(reviewable_path)),
47            )
48
49        client = WebClient("<slack-api-key>")
50
51        ident = '|'.join(
52            [cl['name'] for cl in version['asset']['parent']['link']]
53            + [version['asset']['name'], 'v%03d' % (version['version'])]
54        )
55
56        if reviewable_path:
57            self.logger.info(
58                'Posting Slack message "{}" to channel {}, attaching reviewable "{}"'.format(
59                    ident, self.SLACK_CHANNEL, reviewable_path
60                )
61            )
62            try:
63                response = client.files_upload(
64                    channels=self.SLACK_CHANNEL,
65                    file=reviewable_path,
66                    title=ident,
67                    initial_comment=version['comment'],
68                )
69            finally:
70                os.remove(reviewable_path)  # Not needed anymore
71        else:
72            # Just post a message
73            self.logger.info(
74                'Posting Slack message "{}" to channel {}, without reviewable'.format(
75                    ident, self.SLACK_CHANNEL
76                )
77            )
78            client.chat_postMessage(channel=self.SLACK_CHANNEL, text=ident)
79        if response.get('ok') is False:
80            raise Exception(
81                'Slack file upload failed! Details: {}'.format(response)
82            )
83
84        return {}
85
86
87def register(api_object, **kw):
88    if not isinstance(api_object, ftrack_api.Session):
89        # Exit to avoid registering this plugin again.
90        return
91    plugin = CommonSlackPublisherFinalizerPlugin(api_object)
92    plugin.register()

Breakdown of plugin:

  • With the data argument, the finaliser gets passed on the result from the entire publish process. From this data we harvest the temporary path to thumbnail and asset version id.

  • We transcode the path so we locate the thumbnail copy.

  • A Slack client API session is created

  • An human readable asset version identifier is compiled

  • If a thumbnail were found, it is uploaded to Slack. A standard chat message is posted otherwise.

Add Slack library¶

To be able to use the Slack Python API, we need to add it to our Framework build. We do that by adding the dependency to setup.py:

projects/framework-maya/setup.py

 1..
 2
 3# Configuration.
 4setup(
 5    name='ftrack-framework-maya',
 6    ..
 7    setup_requires=[
 8        ..
 9        'slackclient'
10    ],
11    install_requires=[
12        'slackclient'
13    ],
14    ..
15)

Important

A better approach would be to add the dependency to the framework-core module where the other pipeline dependencies are defined and built.
